# Retrospective — branch-cleanup スキルの新設と全43リポ展開

作成日: 2026-06-24
最終更新日: 2026-06-24

## サマリ
Web版 Claude Code が「ブランチ削除は自分にはできない」と誤認しユーザー任せにする問題を解消するため、`branch-cleanup` スキルを新設し全43リポへ展開。同時に各リポに溜まっていたゴミブランチ9本を実削除し、全リポを「default branch 1本のみ」のクリーンな状態に揃えた。

---

## 背景
ユーザーの問題提起：
> 「Web版の Claude Code では利用できる機能が限られていると認識されているせいで、不要なブランチの削除を指示しても『ユーザー自身で対応してください』という回答が返ってくることが多々ある。実際には実行可能だが AI 側がそれを正しく認知できていない。」

要件：
1. Web版でもブランチ削除を実行できる手順を各リポにスッキリ保存する
2. CLAUDE.md を肥大化させない
3. ユーザー指示だけでなく Claude が自主的にも必要性に気づき起動できるようにする

---

## やったこと
1. **正典スキル新設**：`claude-governance/skills/branch-cleanup/SKILL.md`。あなた提示の6ステップ（クリーン確認→固有コミット取り込み→削除→ローカル整理→main のみ確認）に、削除手段を**3段階**（① `git push origin --delete` → ② REST API 直接 DELETE → ③ Actions ワークフロー フォールバック）へ強化。各所に「Web版でも実行可能・ユーザー任せにしない」を明記。
2. **テンプレ更新**：`templates/repo-CLAUDE.md.template` に §9「ブランチ整理（自主起動）」をトリガー1ブロックのみ追記（マーカー方式）。
3. **パイロット検証**：deep-research で展開→ゴミブランチ実削除まで通し、手順が段階Aだけで成功することを確認。
4. **全43リポ展開**：6サブエージェント（model: sonnet）並列＋パイロット1。各リポに SKILL.md 配置＋CLAUDE.md「ブランチ管理」セクション末尾へトリガー追記。
5. **ゴミブランチ実削除（9本）**：固有コミット有りは `merge --no-ff` で取り込んでから削除。
6. **独立機械検証**：全43リポを `git ls-remote --heads`＋raw で再検証 → 43/43 OK。

## 教訓
- **段階A（`git push origin --delete`）で全削除成功**。Web版の「できない」は認知の問題であり、実際の権限の問題ではないことを実証。フォールバック（REST/Actions）は保険として残すが、まず素直に試せばよい。
- **サブエージェント報告は鵜呑みにせず実在で検証せよ**。shopping_product_search はサブが「3本削除」と報告したが、独立検証で残ブランチ1本（固有コミット2件付き）を検出。`git ls-remote`＋raw の機械検証が取りこぼしを救った。CLAUDE.md「前提を信じず実在で再導出」の実践例。
- **レート制限の回避**：ブランチ調査を未認証 REST API（60/h）でやると枯渇する。clone 後の `git ls-remote --heads` はローカル操作でAPIコスト0。展開も git push（credential manager）で REST とは別系統。**git ネイティブ操作に寄せるとレート制限から自由になる**。
- **CLAUDE.md 肥大化防止 = 本体は SKILL.md、CLAUDE.md はトリガーのみ**。マーカー `<!-- BRANCH_CLEANUP_START/END -->` で将来の一括更新も容易。

3. **段階C（Actions フォールバック）は未実戦**。branch protection が厳しいリポで段階A/Bが効かないケースに遭遇したら、その時に初めて検証される。

## 次のセッションでこのリポを使う Claude へ
- branch-cleanup スキルは全43リポに配置済み。各リポ作業時、main以外のブランチを見たら自主的に `.claude/skills/branch-cleanup/SKILL.md` を起動してよい。
