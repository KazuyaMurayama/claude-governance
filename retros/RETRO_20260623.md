# Retrospective — claude-governance 設立セッション

作成日: 2026-06-23
最終更新日: 2026-06-23

## サマリ
本セッションで `claude-governance` リポを新設し、グローバル運用ルール・設定テンプレ・43リポ共通テンプレ・変更履歴・ブートストラップ手順を一元化した。これにより「整備ルールがセッション会話履歴とローカル `~/.claude/` にしか存在しない」状態を解消した。

---

## 背景
ユーザーの問題提起：
> 「このセッションで進めたようなGitHubリポジトリの整備については、その整備のルール自体を蓄積したり更新したりしているようなGitHubリポジトリはありますか？それとも、整備のルール自体はセッションにしか残っていない状況でしょうか。そうすると、新たなセッションで再現したり呼び出ししたりするのが難しくなる恐れがあります。」

直前まで実施していた整備作業（命名規則 v2.0 への全43リポ展開、モデル使い分け §7 追加、rate limit 対策 env ブロック追加、Dependabot 停止、ブランチ統合）は、すべて以下にしか存在しなかった：
1. ローカル `~/.claude/CLAUDE.md` / `settings.json` / `deliverables-policy.md`
2. 各リポの `CLAUDE.md`（こちらはGitHub上にある）
3. セッション会話履歴（揮発性）

この構造の問題は3つ：
- PCが壊れたら 1 が消失
- Web版 Claude Code から 1 が読めない（claude.ai/code は `~/.claude/` を参照しない）
- 「なぜそのルールにしたか」の経緯が会話履歴にしか残らない

---

## やったこと
1. GitHub REST API で `KazuyaMurayama/claude-governance` リポを新規作成（public）
2. ローカル `C:\Users\user\Desktop\repos\claude-governance\` にディレクトリ構造を作成
   - `global/` — グローバル正典コピー
   - `templates/` — 各リポ共通テンプレ
   - `retros/` — 整備セッションのレトロ
3. ファイル配置
   - `global/CLAUDE.md` ← `~/.claude/CLAUDE.md` の正典コピー
   - `global/deliverables-policy.md` ← `~/.claude/deliverables-policy.md` の正典コピー
   - `global/settings.json.template` ← `~/.claude/settings.json` から機密項目（MCP server PATH、enabledPlugins 等）を除去してテンプレ化
   - `templates/repo-CLAUDE.md.template` ← oogiri CLAUDE.md をベースに placeholder 化した汎用テンプレ
4. メタファイル作成
   - `README.md` — リポの目的・構成・使い方
   - `CLAUDE.md` — 本リポ自身の運用ルール
   - `BOOTSTRAP.md` — 新セッションでの参照手順
   - `CHANGELOG.md` — 本リポの変更履歴＋過去の整備履歴の遡及記録
5. 本ファイル `retros/RETRO_20260623.md` を作成

## 何が良くなったか
| Before | After |
|---|---|
| グローバル設定が PC ローカルのみ | GitHub にバックアップ（公開リポ） |
| Web版 Claude Code でグローバルルールが効かない | Web版でも `KazuyaMurayama/claude-governance` を最初に読ませる運用が可能に |
| 整備の WHY が会話履歴のみ | `CHANGELOG.md` に WHAT/WHY/HOW TO APPLY の3節構成で永続化 |
| 過去の整備履歴が散在 | `CHANGELOG.md` 末尾に遡及記録として集約 |

## まだ未解決・要フォロー
1. **本リポ作成直後の同期はまだ完了していない**
   - `~/.claude/CLAUDE.md` ↔ 本リポ `global/CLAUDE.md` は今この瞬間は一致しているが、今後の更新時に乖離する可能性がある
   - 推奨: グローバル更新時に「本リポへの反映」を必ずチェックリストに入れる
2. **43リポへの「本リポ参照リンク」追加は未実施**
   - 各リポの CLAUDE.md に「上位ガバナンス = `KazuyaMurayama/claude-governance` を参照」の1文を追加すべきか、ユーザー判断待ち
   - 追加すれば各リポ作業時に Claude が自然にこちらを辿れる
3. **`global/settings.json.template` は機密除去版**
   - 完全再現には個人固有値（`mcpServers`、`enabledPlugins`、`extraKnownMarketplaces`）を別途復元する必要がある
   - 別途 personal-secrets リポ（private）を作るか、`~/.claude/settings.local.json` で運用するかは未決
4. **CHANGELOG の遡及記録は会話履歴から再構成したもの**
   - 厳密な日付・順序は不正確な可能性がある（特に2026-06-21、2026-06-22 の境界）
   - 重要なのは「なぜ」の記録なので、日付は近似値として扱う

## 教訓
- **ローカル単独保存の運用ルールは「失われる前提」で扱う**：PC故障・OS再インストール・別環境（Web版）から参照不能 という3つのリスクが常にある
- **WHY を残さない CHANGELOG は git log と同じ**：WHAT は diff から復元できるが、WHY は誰も復元できない
- **「self-contained」と「上位ガバナンス参照」は両立する**：各リポ CLAUDE.md は単独で動作可能であるべきだが、上位ガバナンスへのリンクを添えることで「同期不足を検知する手がかり」を提供できる

## 次のセッションでこのリポを使う Claude へ
1. `BOOTSTRAP.md` を読む
2. `CHANGELOG.md` で最新の変更経緯を確認
3. もし `~/.claude/CLAUDE.md` と `global/CLAUDE.md` に差分があれば、ユーザーに同期方向を確認してから片寄せ
4. 整備セッションを実施したら、必ず `CHANGELOG.md` を更新し、大きな変更なら `retros/RETRO_YYYYMMDD.md` を残す
