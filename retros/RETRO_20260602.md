## 学んだこと・教訓

### スキルトリガー設計
- トリガー条件は「作業の自然な発話」に合わせる必要がある
- 「コミット前のみ」は狭すぎ。「QC・レビューフェーズ」も明示しないと起動しない
- リポタイプ別の分類（A/B/C/D）は有効。全リポ同一ルールではなく用途別に最小必要スキルを設定

### ブランチ管理
- 「ブランチを作らない」だけでは不十分。「作った場合の完了条件」も明文化が必要
- Claude Code Web版は自動でブランチを生成する場合がある → セッション終了前のマージを強制

### スクリプト運用
- Desktop禁止後の一時スクリプト置き場: `C:\Users\user\AppData\Local\Temp\`
- node -e でのバックティック・全角文字 → スクリプトファイル化が確実
